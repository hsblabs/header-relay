Header Relay

プライバシーポリシー

最終更新日: 2026年7月22日

目的

本ポリシーはHeader Relay Chrome拡張機能とドキュメントサイトを対象とします。拡張機能では、固定ヘッダーまたは保持したヘッダーを付与した際のブラウザ上のHTTPリクエストを確認できます。

ホスト権限

既定のローカル開発用途としてhttp://localhost/*へのアクセス権限を含みます。それ以外のホスト権限は任意であり、対象オリジンを追加したときだけ要求します。ヘッダーの取得と付与は、有効なプロファイルの対象オリジンと除外パスに限定されます。

ローカルに保存するデータ

プロファイル設定とランダム生成したUUID v7の分析用client_idは、拡張機能用のローカルストレージに保存します。取得したヘッダー値と実行時のセッション状態は、メモリ上のセッションストレージだけに保持します。診断用の監査ログはIndexedDBに保存しますが、ヘッダー値とリクエストURLは永続保存しません。取得した値はデフォルトでマスクし、明示的な表示またはコピー操作でのみ確認できます。

利用目的の限定

ブラウザ権限とローカル保存データは、設定したレスポンスヘッダー値を取得し、利用者が選んだ対象オリジンへのリクエストに設定済みヘッダーを中継するという単一の目的にのみ利用します。広告、信用評価、融資、その他の目的には利用しません。

データ共有

プロファイル設定、監査ログ、利用イベント、分析用client_id、閲覧データ、取得したヘッダーを、開発者、分析事業者、無関係なサーバーへ送信しません。設定したヘッダー値は、利用者が選択した対象オリジンへの一致するリクエストにだけ付与します。

人によるアクセスなし

拡張機能を通じて、開発者がプロファイル設定、取得した値、監査ログ、利用イベント、分析用client_id、閲覧履歴を受領、閲覧、販売、共有することはありません。ローカルに保存したデータはChromeプロファイル内に留まり、設定したヘッダー値は一致する対象オリジンにだけ開示します。

Webサイトのアクセス解析

同意した場合に限り、このドキュメントサイトはページ閲覧と利用状況の集計にGoogle Analytics 4（GA4）を使用します。GA4はページURL、参照元、おおよその地域、ブラウザと端末の情報を処理し、hrから始まるファーストパーティのアクセス解析Cookieを設定する場合があります。このCookieの対象は本サイトの/header-relayパスに限定します。Google Signalsと広告パーソナライズは無効です。拒否した場合はタグを読み込みません。選択結果はlocalStorageに保存し、フッターの「アクセス解析の設定」から変更できます。拡張機能のプロファイル、設定または取得したヘッダー値、監査ログ、閲覧履歴をGA4へ送信することはありません。

拡張機能の分析

Production buildはローカルの分析用client_idを生成しますが、利用イベントを出力せず、分析データを外部送信しません。pnpm devで起動した開発ビルドだけが、機密情報を含まない型付きイベントをService Workerのコンソールへ出力します。拡張機能に外部分析を追加する場合は、リリース前に本ポリシーとChrome Web Storeの申告を更新します。

監査ログの保持

診断用の監査ログは最新1,000件に制限され、7日を過ぎた項目は自動的に削除されます。リクエストURLは監査ログへ永続保存しません。現在のブラウザセッション中だけ、直近ログの表示用にオリジンとパス名を保持します。管理画面からすべての監査ログを削除できます。

利用者による制御

プロファイルの無効化、対象オリジンの削除、保持した値の消去、拡張機能のアンインストールにより、取得と付与の動作を停止できます。
