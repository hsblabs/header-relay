고정 및 추적 Cookie FAQ

Cookie 추적, 교체, 보관 기간, 충돌, 권한과 기존 설정 마이그레이션을 설명합니다.

무엇을 설정할 수 있나요?

프로필마다 고정 Cookie와 추적 Cookie를 설정합니다. 고정 Cookie에는 항상 보낼 이름과 값을, 추적 Cookie에는 Set-Cookie 응답에서 가져올 이름을 등록합니다.

추적 Cookie는 어떻게 갱신되나요?

Header Relay는 활성 프로필의 대상 응답에서 등록된 Cookie 이름의 값만 저장합니다. 같은 응답에 같은 이름이 여러 번 있으면 마지막 정상 갱신 또는 삭제를 적용합니다.

Cookie 삭제 응답은 어떻게 처리하나요?

유효한 Max-Age가 0 이하이면 추적값을 삭제합니다. Max-Age가 없을 때는 과거 Expires도 삭제로 처리합니다. 둘 다 있으면 Max-Age가 우선하며, 잘못된 Set-Cookie 하나가 정상 항목을 막지 않습니다.

Chrome이 관리하는 Cookie는 어떻게 되나요?

Header Relay는 Chrome Cookie 저장소를 변경하지 않습니다. 관리 값이 하나 이상 적용되면 요청의 Cookie 헤더를 완전히 교체하고, 후보가 없으면 Chrome의 원래 Cookie 헤더를 변경하지 않습니다.

Domain, Path, Secure, HttpOnly, SameSite를 재현하나요?

재현하지 않습니다. Cookie 이름과 값을 추적하고 Max-Age와 Expires는 삭제 판단에만 사용합니다. 전송 범위는 프로필의 Target Origin과 Excluded Path로 정합니다.

고정 Cookie와 추적 Cookie 이름이 같으면 어떻게 되나요?

같은 프로필에서는 고정 Cookie가 우선합니다. Cookie 이름은 대소문자를 구분하므로 SID와 sid는 다른 이름입니다.

여러 프로필이 같은 요청에 일치하면 어떻게 되나요?

서로 다른 Cookie 이름은 하나의 헤더로 결합합니다. 같은 이름이 여러 활성 프로필에 있으면 충돌로 처리하고 값을 임의로 선택하지 않으며 Header Relay의 관리 DNR 규칙을 중지합니다.

Excluded Path에서는 어떻게 되나요?

일치한 Excluded Path에서는 해당 프로필의 Cookie를 전송하지 않습니다. 안전하게 결합할 수 있으면 다른 일치 프로필의 Cookie는 남깁니다.

Cookie 값은 어디에서 확인하나요?

관리 화면, Popup, URL Probe에서 확인합니다. 추적값은 개별 또는 프로필 단위로 지울 수 있습니다.

추적 Cookie는 언제까지 보관하나요?

현재 브라우저 세션에만 보관합니다. 브라우저나 확장 프로그램 재시작, 프로필 비활성화, Cookie 또는 Target Origin 변경, 호스트 권한 취소, 수동 삭제 시 제거합니다.

Cookie 이름과 값이 로그에 저장되나요?

감사 로그에는 파싱된 Cookie 이름과 처리 결과가 포함될 수 있지만 값과 원본 Set-Cookie 문자열은 기록하지 않습니다. 분석 이벤트에는 이름, 값, Target Origin, 입력 문자열을 기록하지 않습니다.

기존 범용 Cookie 헤더 설정은 어떻게 되나요?

손실 없이 변환 가능한 Fixed Header Cookie는 전용 고정 Cookie로 옮깁니다. 변환할 수 없는 설정과 추적 이름을 추측할 수 없는 Captured Header Cookie는 검토용 마이그레이션 문제로 보존합니다.

새 Chrome 권한이 필요한가요?

필요하지 않습니다. cookies 권한을 요청하지 않고 기존 webRequest, declarativeNetRequest와 Target Origin별 호스트 권한을 사용합니다.
