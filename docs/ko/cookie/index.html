Cookie 값을 수동으로 복사하지 않는 고정 및 추적 Cookie.

Header Relay는 로컬, 스테이징, 사내 환경의 API 테스트를 위해 Cookie 요청 헤더를 전용 기능으로 관리합니다.

도쿄, 2026년 7월 22일

요청 사이에서 Cookie 값을 복사하지 않기

Cookie 인증 API를 테스트할 때 개발자는 Set-Cookie를 확인하고 다음 요청에 값을 옮겨 적곤 합니다. 로그인이나 세션이 갱신될 때 작업이 반복되고, 오래된 값은 테스트 결과를 잘못 해석하게 만들 수 있습니다.

새 Cookie 기능에서는 프로필마다 고정 Cookie와 추적할 Cookie 이름을 등록합니다. Header Relay는 대상 오리진의 Set-Cookie에서 등록된 값을 가져와 이후 일치하는 요청에 전송합니다.

최종 Cookie 헤더를 하나로 결합

고정값과 추적값은 순서가 결정된 하나의 Cookie 헤더로 결합됩니다. 같은 프로필 안에서 이름이 겹치면 고정 Cookie가 우선합니다. 여러 프로필의 이름이 다르면 결합하고, 같으면 임의로 선택하지 않고 충돌로 알립니다.

URL Probe에서 요청을 보내기 전에 일치한 프로필과 최종 Cookie 헤더를 확인할 수 있습니다. 관리 화면과 Popup에서도 현재 값을 보고 추적값을 개별 또는 프로필 단위로 지울 수 있습니다.

Chrome Cookie 저장소와 분리

Header Relay는 Chrome Cookie 저장소를 읽거나 변경하지 않으며 cookies 권한도 요청하지 않습니다. 관리 값이 하나 이상 적용될 때만 해당 요청의 Cookie 헤더를 교체합니다. 전송 후보가 없으면 Chrome이 만든 Cookie 헤더를 그대로 둡니다.

추적값은 현재 브라우저 세션에만 보관됩니다. 브라우저나 확장 프로그램 재시작, 프로필 비활성화, 관련 설정 변경, 권한 취소 또는 수동 삭제 시 제거됩니다.

Cookie 테스트를 위한 세 가지 기능

고정 Cookie

활성 프로필에서 항상 전송할 Cookie 이름과 값을 설정합니다.

추적 Cookie

Cookie 이름을 등록하고 일치하는 Set-Cookie 응답에서 세션 값을 갱신합니다.

전송 전 확인

관리 화면, Popup, URL Probe에서 현재 값과 최종 Cookie 헤더를 확인합니다.

명확하게 유지하는 경계

  • 활성 프로필, 설정된 대상 오리진, 제외되지 않은 경로만 대상으로 합니다.
  • Cookie 속성의 브라우저 전송 조건은 재현하지 않고 Target Origin과 Excluded Path로 범위를 정합니다.
  • Cookie 값과 원본 Set-Cookie 문자열을 감사 로그나 분석 이벤트에 기록하지 않습니다.
  • 변환할 수 없는 기존 Cookie 헤더 설정은 마이그레이션 문제로 보존합니다.
