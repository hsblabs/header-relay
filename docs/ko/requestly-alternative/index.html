Chrome 확장 프로그램 비교

헤더 기능만 필요하다면, 더 가벼운 Requestly 대안을 찾고 계신가요?

Requestly는 30만 명 이상의 개발자가 사용하는 오픈소스 HTTP 인터셉터로, 헤더 수정, API 모킹, URL 리디렉션, 응답 재정의, 스크립트 삽입을 하나의 확장 프로그램에서 제공합니다. Header Relay는 그중 한 부분, 즉 선택한 응답 헤더를 캡처하고 고정값 또는 캡처한 값을 일치하는 요청에 전달하는 기능만 제공합니다. 헤더 관련 기능만 사용하신다면, 더 좁은 범위의 도구가 권한도 적고 검토할 범위도 작습니다.

실제로 다른 점.

안전성에 대한 지적이 아닙니다 — Requestly는 오픈소스이며 널리 사용됩니다. 이것은 범위의 트레이드오프입니다: 모든 것을 하는 도구인가, 한 가지만 하는 도구인가.

단일 목적

Header Relay는 헤더 캡처와 전달만 수행합니다. Requestly의 모킹, 리디렉션, 스크립트 삽입에 해당하는 기능은 없습니다 — 그만큼 잘못 설정할 여지도 없습니다.

작고 검토하기 쉬운 코드

헤더 수명 주기 로직은 Chrome의 declarativeNetRequest API와 로컬 저장소만 사용하는 하나의 모듈에 담겨 있습니다.

캡처한 값을 숨기지 않음

캡처한 헤더 값은 팝업과 관리 화면에 일반 텍스트로 표시되며 추상화 뒤에 숨지 않습니다.

계정 불필요, 클라우드 동기화 없음

프로필과 캡처한 값은 로컬 Chrome 저장소에 남습니다. 로그인할 것이 없습니다.

헤더 기능만 사용했다면, 전환은 3단계입니다.

Header Relay는 다른 확장 프로그램의 규칙을 가져오지 않습니다 — 헤더 전용 구성이라면 금방 다시 만들 수 있습니다.

01

프로필 만들기

헤더 규칙을 적용하던 대상 오리진을 추가하세요.

02

고정 헤더 옮기기

API 키, 클라이언트 ID 같은 고정값은 Fixed Header로 설정합니다.

03

캡처 값 자동화하기

세션 토큰이나 트레이스 ID처럼 응답에서 추출하던 값은 Captured Header로 설정하면 자동으로 다시 첨부됩니다.

Requestly에서 전환하는 것에 대한 자주 묻는 질문.

Requestly가 안전하지 않은가요?

아닙니다. 이 페이지는 Requestly의 문제를 보고하는 것이 아닙니다. Requestly는 오픈소스이며 널리 사용됩니다. 여기서의 비교는 순전히 범위 차이에 관한 것입니다 — Header Relay는 더 좁은 역할만 담당합니다.

무엇을 포기하게 되나요?

API 모킹, 응답 본문 재정의, URL 리디렉션, 스크립트 삽입입니다. Header Relay는 이 기능들을 제공하지 않습니다. 이런 기능이 필요하다면 Requestly가 더 적합합니다.

둘 다 동시에 사용할 수 있나요?

일반적으로 가능합니다. Header Relay는 활성화된 프로필의 대상 오리진에 대한 헤더만 다루므로, 두 확장 프로그램이 동일한 요청의 동일한 헤더를 대상으로 하지 않는 한 충돌하지 않습니다.
