Header Relay

개인정보처리방침

최종 업데이트: 2026년 7월 2일

목적

Header Relay는 API 개발과 테스트를 위한 Chrome 확장 프로그램입니다. 고정 또는 캡처한 헤더를 추가했을 때 브라우저 기반 HTTP 요청의 동작을 확인할 수 있습니다.

호스트 권한

개발자가 로컬, 스테이징, 사내 또는 테스트 오리진을 선택할 수 있도록 모든 URL에 대한 접근 권한을 요청합니다. 헤더 캡처와 추가는 활성화된 프로필의 대상 오리진과 제외 경로로 제한됩니다.

로컬에 저장되는 데이터

프로필, 캡처한 헤더 값, 세션 상태, 감사 로그는 확장 프로그램의 브라우저 저장소와 IndexedDB에 로컬로 저장됩니다. 요청에 실제로 사용되는 값을 확인할 수 있도록 캡처한 값은 팝업과 관리 화면에 일반 텍스트로 표시됩니다.

제한된 사용

브라우저 권한과 로컬 저장 데이터는 설정한 응답 헤더 값을 캡처하고 개발자가 선택한 대상 오리진에 설정된 요청 헤더를 전달하는 단일 목적에만 사용됩니다. 광고, 신용 평가, 대출 또는 관련 없는 목적으로 사용하지 않습니다.

데이터 공유

프로필 설정, 캡처한 헤더, 감사 로그 또는 탐색 데이터를 외부 서버로 전송하지 않습니다. 네트워크 요청은 설정한 프로필을 사용하는 동안 사용자가 여는 페이지와 API에 대한 요청뿐입니다.

사람의 접근 없음

개발자는 프로필 설정, 캡처한 값, 감사 로그 또는 탐색 활동을 수신, 열람, 판매하거나 공유하지 않습니다. 사용자가 직접 내보내거나 공개하지 않는 한 모든 데이터는 로컬 Chrome 프로필에 남습니다.

감사 로그 보관

감사 로그는 최근 1,000개로 제한되며 7일이 지난 항목은 자동으로 삭제됩니다. 기록되는 URL에는 오리진과 경로명만 포함되고 쿼리 문자열과 프래그먼트는 제외됩니다. 관리 화면에서 모든 감사 로그를 삭제할 수 있습니다.

사용자 제어

프로필 비활성화, 대상 오리진 제거, 캡처 값 삭제 또는 확장 프로그램 제거를 통해 캡처와 추가 동작을 중지할 수 있습니다.
