Header Relay

개인정보처리방침

최종 업데이트: 2026년 7월 17일

목적

Header Relay는 API 개발과 테스트를 위한 Chrome 확장 프로그램입니다. 고정 또는 캡처한 헤더를 추가했을 때 브라우저 기반 HTTP 요청의 동작을 확인할 수 있습니다.

호스트 권한

기본 로컬 개발 워크플로를 위해 http://localhost/* 액세스 권한이 포함됩니다. 그 외 호스트 권한은 선택 사항이며 대상 오리진을 추가할 때만 요청합니다. 헤더 캡처와 추가는 활성화된 프로필의 대상 오리진과 제외 경로로 제한됩니다.

로컬에 저장되는 데이터

프로필 설정과 무작위로 생성한 UUID v7 분석 client_id는 확장 프로그램의 로컬 저장소에 저장됩니다. 캡처한 헤더 값과 런타임 세션 상태는 메모리의 세션 저장소에만 유지됩니다. 진단 감사 로그는 IndexedDB를 사용하지만 헤더 값이나 요청 URL을 영구 저장하지 않습니다. 캡처한 값은 기본적으로 마스킹되며 명시적인 표시 또는 복사 작업으로만 확인할 수 있습니다.

제한된 사용

브라우저 권한과 로컬 저장 데이터는 설정한 응답 헤더 값을 캡처하고 개발자가 선택한 대상 오리진에 설정된 요청 헤더를 전달하는 단일 목적에만 사용됩니다. 광고, 신용 평가, 대출 또는 관련 없는 목적으로 사용하지 않습니다.

데이터 공유

프로필 설정, 감사 로그, 사용 이벤트, 분석 client_id, 탐색 데이터 또는 캡처한 헤더를 개발자, 분석 제공업체 또는 관련 없는 서버로 전송하지 않습니다. 설정한 헤더 값은 사용자가 선택한 대상 오리진에 대한 일치 요청에만 추가됩니다.

사람의 접근 없음

개발자는 프로필 설정, 캡처한 값, 감사 로그, 사용 이벤트, 분석 client_id 또는 탐색 활동을 수신, 열람, 판매하거나 공유하지 않습니다. 로컬에 저장된 데이터는 Chrome 프로필에 남으며 설정한 헤더 값은 일치하는 대상 오리진에만 공개됩니다.

분석

Production build는 로컬 분석 client_id를 생성하지만 사용 이벤트를 출력하거나 분석 데이터를 외부로 전송하지 않습니다. pnpm dev로 시작한 개발 빌드만 민감한 정보가 없는 제한된 형식의 이벤트를 Service Worker 콘솔에 출력합니다. 외부 분석을 추가하려면 릴리스 전에 이 정책과 Chrome Web Store 공개 내용을 업데이트합니다.

감사 로그 보관

진단 감사 로그는 최근 1,000개로 제한되며 7일이 지난 항목은 자동으로 삭제됩니다. 요청 URL은 감사 로그에 영구 저장되지 않습니다. 현재 브라우저 세션에서는 최근 로그 표시를 위해 오리진과 경로명만 유지합니다. 관리 화면에서 모든 감사 로그를 삭제할 수 있습니다.

사용자 제어

프로필 비활성화, 대상 오리진 제거, 캡처 값 삭제 또는 확장 프로그램 제거를 통해 캡처와 추가 동작을 중지할 수 있습니다.
