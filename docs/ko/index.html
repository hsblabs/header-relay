API 개발용 Chrome 확장 프로그램

브라우저 테스트에 필요한 헤더를 전달합니다.

Header Relay는 선택한 응답 헤더를 저장하고, 지정한 오리진에 대한 이후 요청에 고정값 또는 저장된 값을 추가합니다.

Chrome에 추가개인정보처리방침
profile / local-api / enabled
대상 오리진의 응답
x-session-token: dev-session-42
다음 일치 요청
x-client-id: header-relay
x-session-token: dev-session-42
이 대상 오리진에서 활성화됨

제어 가능한 요청 흐름.

대상 오리진과 사용할 헤더를 설정하고 애플리케이션 코드를 변경하지 않은 채 세션 상태를 확인할 수 있습니다.

01

범위 선택

헤더 캡처와 추가는 활성화된 프로필과 설정된 대상 오리진에만 적용됩니다.

02

값 캡처

선택한 응답 헤더 값은 확장 프로그램의 로컬 세션 상태에 저장됩니다.

03

헤더 전달

고정 헤더와 캡처한 헤더를 이후 일치하는 요청에 추가합니다. 제외 경로는 변경하지 않습니다.

로컬 우선 설계.

로컬, 스테이징, 사내 및 테스트 환경을 위한 확장 프로그램입니다. 데이터 처리는 필요한 범위로 제한됩니다.

확장 프로그램 데이터는 로컬에 유지

프로필, 세션 상태, 감사 로그를 개발자나 관련 없는 데이터 서비스로 전송하지 않습니다. 설정한 헤더 값은 일치하는 대상 오리진에 대한 요청에만 추가됩니다.

명확한 보관 제한

진단 감사 로그는 최근 1,000개와 최대 7일로 제한되며 언제든 삭제할 수 있습니다.

URL을 영구 저장하지 않음

요청 URL은 영구 감사 로그에 기록되지 않습니다. 현재 브라우저 세션에서는 최근 로그 표시를 위해 오리진과 경로명만 유지합니다.

사용자가 제어

프로필 비활성화, 대상 오리진 제거, 캡처 값 삭제 또는 확장 프로그램 제거로 동작을 중지할 수 있습니다.

기본 로컬 개발 워크플로를 위해 HTTP localhost 액세스가 포함됩니다. 그 외 호스트는 대상 오리진을 추가할 때만 요청합니다. 캡처와 추가는 활성화된 프로필에 설정한 오리진으로 제한됩니다.

새 기능: 고정 및 추적 Cookie로 Cookie 인증 API 테스트를 반복하기 쉬워졌습니다.

Header Relay에 대해 자주 묻는 질문.

API 개발 워크플로에서 Header Relay를 평가하는 데 필요한 핵심 정보입니다.

Header Relay는 어떤 기능을 하나요?

선택한 응답 헤더를 저장하고 활성화된 프로필과 일치하는 이후 브라우저 요청에 고정값 또는 저장된 헤더 값을 추가합니다.

어떤 요청을 변경할 수 있나요?

활성화된 프로필에 설정된 대상 오리진과 일치하는 요청만 대상이 됩니다. 제외 경로는 변경하지 않습니다.

데이터를 외부 서버로 전송하나요?

확장 프로그램은 개발자, 분석 제공업체 또는 관련 없는 서버로 데이터를 전송하지 않습니다. 설정한 헤더 값은 일치하는 대상 오리진에 대한 요청에만 추가되며 프로필, 세션 상태, 감사 로그는 로컬 Chrome 프로필에 남습니다. 이와 별도로 문서 사이트는 동의한 후에만 GA4를 사용합니다.

어떤 환경에서 사용해야 하나요?

로컬, 스테이징, 사내 환경과 같은 통제된 환경에서 API를 개발하고 테스트하는 용도로 설계되었습니다.
